¿Cómo reconstruir las capas 2.5D?

El motor ya funciona — buildTiledLayers() sabe leer props_back / props_front.
El problema: los mapas no tienen esas capas. Hay que dárselas.

Situación actual vs. contrato 2.5D
❌ Ahora — comisaria-sala-espera.json
Objects3 objetos (exits, NPCs)
Walls193 tiles (solo borde)
Floor⚠ 0 tiles pintados

buildTiledLayers() procesa Floor y Walls, pero sin props_back ni props_front no hay Y-sort para mobiliario. El suelo aparece negro (procedural) porque Floor está vacío.

Elige el enfoque de implementación
A

Script Python → mapa completo con contenido real ⭐ Recomendado

Un script genera la nueva versión de comisaria-sala-espera.json con todas las capas 2.5D y contenido pintado: suelo con tiles de tiny-dungeon, muebles en props_back (escritorios, sillas) y una barra de recepción en props_front que tapa al jugador cuando camina hacia el fondo. Los NPCs y exits del Objects layer se conservan.

✅ Pros
• Causa raíz resuelta en datos
• Y-sort real con tiles
• Extensible a los otros 13 mapas
⚠ Consideraciones
• Hay que mapear GIDs del tileset
• ~2-3h de trabajo
B

Añadir capas vacías al JSON + arreglar depth procedural

El script solo añade las capas props_back, props_front, floor al JSON existente (con tiles vacíos o mínimos). Paralelamente, se corrige renderComisariaEnvironment() para que use actorDepth(y) en lugar de depth fijo 7/8. El mobiliario sigue siendo procedural pero ahora se ordena por Y.

✅ Pros
• Rápido (~1h)
• Mejora visible inmediata
⚠ Contras
• Props siguen siendo procedurales
• Resolución parcial del P0
• Deuda de autoría pendiente
C

A + diseño completo de sala de espera con atlas Kenney

Todo lo de la Opción A, pero con un diseño más elaborado: filas de sillas de espera, macetas, pósters en la pared posterior, ventana con barras, suelo con variación de tiles, counter de recepción como props_front sólido. Resultado: la sala de espera se ve como sala de espera de comisaría, no como un cuarto genérico.

✅ Pros
• Máxima calidad visual
• Muestra el potencial real
⚠ Consideraciones
• ~4-5h de trabajo
• Necesita referencia del tileset

En todos los casos el escenario objetivo es comisaria-sala-espera — ya tiene NPCs con characterId asignado (R6), lo que hace la demo más completa.
